Notas generales

En esta sección se describen las métricas usadas para la realización del presente Benchmark y la memoria técnica.

Para mejorar la claridad del documento, estas han sido agrupadas, de forma que se pueda establecer un peso denominado de valoración cada uno de los grupos, y para cada una de las métricas perteneciente a un mismo grupo de forma individual.

Los pesos son ponderados (tanto para el grupo como individualmente para cada métrica dentro de un grupo, de forma que la suma de pesos para todos los grupos será 1, y la de todas las métricas individuales de un mismo grupo, también será 1.

Por otro lado, tanto para métricas individuales, como pora las de grupo se creara un peso de incertidumbre que será aplicable en caso de que no exista valor para un determinado Triple Store, ya sea en una métrica individual o en una métrica de grupo.

Como resultado final se ofrecerá una benchmark para todos los Triple Stores, ofreciendo su grado de valoración y el grado incertidumbre de forma agrupada.

Todos los pesos de valoración pueden ser configurados

   SP2B
Descripción

Benchmark para estimar optimación, generalidad y beneficios en el mundo real realizado por la Universidad de Friburgo. Genera un conjunto de datos, con documentos arbitrariamente grandes con el formato de la biblioteca DBLP (Base de datos en formato triple store sobre ciencias de la computación), y realiza consultas sobre ellos para medir el rendimiento. El generador de datos usa como modelo la base de datos DBLP, ya que tiene muchas características de los datos en el mundo real, como por ejemplo el carácter social, e imita las correlaciones que existen en ella sobre entidades, tales como artículos, autores...., para crear los datos se analizan las frecuencias e interacciones de las entidades, de forma que el resultado es los más fiel posible.
